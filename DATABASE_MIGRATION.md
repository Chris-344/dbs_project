# Database Migration Guide

## Prerequisites
- Oracle Database with active connection
- SQL*Plus or SQLDeveloper access
- Connection credentials configured in `.env`

---

## Migration Steps

### Step 1: Backup Current Database (Optional but Recommended)

```sql
-- Check existing data
SELECT COUNT(*) as author_count FROM authors;
SELECT COUNT(*) as publication_count FROM publications;
```

### Step 2: Update PUBLICATIONS Table

Add the `COPIES_AVAILABLE` column:

```sql
ALTER TABLE PUBLICATIONS ADD (COPIES_AVAILABLE NUMBER DEFAULT 1);
```

Verify the column was added:

```sql
DESC PUBLICATIONS;
```

### Step 3: Create BORROWING Table

```sql
CREATE TABLE BORROWING (
    BORROW_ID NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    PUB_ID NUMBER REFERENCES PUBLICATIONS(PUB_ID),
    BORROWER_NAME VARCHAR2(100) NOT NULL,
    ISSUE_DATE DATE NOT NULL,
    RETURN_DATE DATE NOT NULL,
    RETURNED NUMBER DEFAULT 0,
    CREATED_AT TIMESTAMP DEFAULT SYSDATE
);
```

### Step 4: Create Indexes (For Performance)

```sql
CREATE INDEX idx_borrowing_pubid ON BORROWING(PUB_ID);
CREATE INDEX idx_borrowing_returndate ON BORROWING(RETURN_DATE);
```

### Step 5: Create Borrowing Procedure

```sql
CREATE OR REPLACE PROCEDURE BORROW_PUBLICATION(
    p_pub_id IN NUMBER,
    p_borrower_name IN VARCHAR2,
    p_issue_date IN DATE,
    p_return_date IN DATE,
    p_borrow_id OUT NUMBER
) AS
BEGIN
    INSERT INTO BORROWING (PUB_ID, BORROWER_NAME, ISSUE_DATE, RETURN_DATE)
    VALUES (p_pub_id, p_borrower_name, p_issue_date, p_return_date)
    RETURNING BORROW_ID INTO p_borrow_id;
END;
/
```

### Step 6: Create Availability Function

```sql
CREATE OR REPLACE FUNCTION GET_PUBLICATION_AVAILABILITY(p_pub_id IN NUMBER)
RETURN NUMBER AS
    v_copies_available NUMBER;
    v_active_borrows NUMBER;
BEGIN
    SELECT COPIES_AVAILABLE INTO v_copies_available 
    FROM PUBLICATIONS 
    WHERE PUB_ID = p_pub_id;
    
    SELECT COUNT(*) INTO v_active_borrows
    FROM BORROWING
    WHERE PUB_ID = p_pub_id 
    AND RETURNED = 0
    AND TRUNC(RETURN_DATE) >= TRUNC(SYSDATE);
    
    RETURN v_copies_available - v_active_borrows;
END;
/
```

### Step 7: Verify Installation

```sql
-- Check BORROWING table exists
DESC BORROWING;

-- Check procedure exists
SELECT object_name, object_type 
FROM user_objects 
WHERE object_name = 'BORROW_PUBLICATION';

-- Check function exists
SELECT object_name, object_type 
FROM user_objects 
WHERE object_name = 'GET_PUBLICATION_AVAILABILITY';

-- Test function
SELECT GET_PUBLICATION_AVAILABILITY(1) FROM DUAL;
```

## Rollback (If Needed)

### Drop Borrowing-Related Objects

```sql
-- Drop function
DROP FUNCTION GET_PUBLICATION_AVAILABILITY;

-- Drop procedure
DROP PROCEDURE BORROW_PUBLICATION;

-- Drop table
DROP TABLE BORROWING;

-- Drop indexes (if added)
DROP INDEX idx_borrowing_pubid;
DROP INDEX idx_borrowing_returndate;

-- Remove column from PUBLICATIONS
ALTER TABLE PUBLICATIONS DROP COLUMN COPIES_AVAILABLE;
```

## Sample Data Setup

After migration, populate initial data:

```sql
-- Set copies available for existing publications
UPDATE PUBLICATIONS SET COPIES_AVAILABLE = 2 WHERE PUB_ID = 1;
UPDATE PUBLICATIONS SET COPIES_AVAILABLE = 1 WHERE PUB_ID = 2;
UPDATE PUBLICATIONS SET COPIES_AVAILABLE = 3 WHERE PUB_ID = 3;

-- Or update all at once
UPDATE PUBLICATIONS SET COPIES_AVAILABLE = 1;

-- Verify
SELECT PUB_ID, TITLE, COPIES_AVAILABLE FROM PUBLICATIONS;
```

## Troubleshooting

### Issue: "ORA-00955: name is already used by an existing object"
- **Solution**: The table/procedure already exists. Drop and recreate, or check existing objects.

### Issue: "ORA-02291: integrity constraint violated"
- **Solution**: PUB_ID doesn't exist in PUBLICATIONS. Insert publication first.

### Issue: Availability always returns NULL
- **Solution**: Ensure COPIES_AVAILABLE column was added correctly. Run `DESC PUBLICATIONS;`

---

## Best Practices

1. **Always backup** before running migrations on production
2. **Test in development** first
3. **Commit changes** after successful verification
4. **Monitor indexes** for query performance
5. **Archive old borrowing records** periodically (not implemented)

---

## Notes

- No data loss occurs during this migration
- Existing authors and publications remain intact
- Add default COPIES_AVAILABLE = 1 for existing publications
- Return date validation is client-side; ensure valid date ranges
